Навігація та перегляд файлів
pwd — показати поточну директорію («де я зараз»).  
ls — список файлів і папок.  
  `ls -l` — детальний список, `ls -a` — показати приховані файли, `ls -la` — обидва разом.  
cd папка — перейти в папку.  
  `cd ..` — повернутися на рівень вище, `cd ~` — у домашню папку, `cd -` — у попередню.  
mkdir назва — створити папку.  
rmdir назва — видалити порожню папку.  
rm файл — видалити файл.  
  `rm -r папка` — видалити папку разом із вмістом, `rm -f файл` — примусово без запитань.  
cp джерело призначення — копіювати файл/папку.  
mv джерело призначення — перемістити або перейменувати файл/папку.

Робота з текстом та вмістом файлів
cat файл — вивести весь вміст файлу одразу.  
less файл — переглядати файл посторінково (вихід – `q`).  
head файл / tail файл — показати перші / останні 10 рядків (кількість можна змінити: `head -n 5`).  
nano файл — простий текстовий редактор у терміналі (Ctrl+X для виходу, Y/N для збереження).  
echo "текст" — вивести текст на екран (часто використовують для запису у файл: `echo "рядок" > файл`).

Права, користувачі, sudo
whoami — показати поточного користувача.  
id — показати UID, GID та групи користувача.  
sudo команда — виконати команду з правами root.  
su - — перемкнутися на root (запитає пароль root).  
chmod — змінити права доступу до файлу/папки.  
  Наприклад: `chmod +x script.sh` — зробити файл виконуваним.  
chown — змінити власника файлу (тільки з sudo).  
passwd — змінити свій пароль; `sudo passwd ім’я` — змінити пароль іншого користувача.

Інформація про систему
fastfetch / neofetch — красива інформація про систему. Якщо не встановлено: `sudo pacman -S fastfetch`.  
uname -a — версія ядра та архітектура.  
lsb_release -a — назва та версія дистрибутиву (якщо є lsb-release).  
df -h — вільне місце на дисках.  
free -h — оперативна пам’ять (використано/вільно).  
htop (або `top`) — інтерактивний диспетчер процесів (якщо немає htop: `sudo pacman -S htop`).  
uptime — скільки часу працює система.  
hostnamectl — повна інформація про хост (на systemd-системах, як Arch).

Керування пакетами (Arch / AUR)
sudo pacman -Syu — оновити всю систему.  
sudo pacman -S назва — встановити пакет з офіційних репозиторіїв.  
sudo pacman -Rns назва — повністю видалити пакет із залежностями.  
sudo pacman -Ss ключове_слово — шукати пакет.  
yay -S назва.bin — встановити пакет з AUR (використовує бінарний кеш).  
  Якщо yay ще не встановлено:  
  sudo pacman -S --needed git base-devel && git clone https://aur.archlinux.org/yay-bin.git && cd yay-bin && makepkg -si

Робота з процесами
ps aux — список усіх запущених процесів.  
kill PID — завершити процес за його ідентифікатором (PID дізнаються з `ps` або `htop`).  
killall назва — завершити всі процеси з таким іменем.  
Ctrl+C — перервати виконання поточної програми в терміналі.  
Ctrl+Z — призупинити програму, потім `bg` (продовжити у фоні) або `fg` (повернути на передній план).

Мережа та інтернет
ping сайт — перевірити з’єднання (Ctrl+C для зупинки).  
ip a — показати IP-адреси мережевих інтерфейсів.  
curl URL — завантажити вміст сторінки (або файл).  
wget URL — завантажити файл із мережі.

Архіви та компресія
tar -xf архів.tar.gz — розпакувати .tar.gz.  
tar -czf назва.tar.gz папка — запакувати папку в tar.gz.  
unzip архів.zip — розпакувати .zip (якщо unzip не встановлено: `sudo pacman -S unzip`).

дрібниці

history — показати історію команд.  !номер — виконати команду з історії за номером.  
alias — створити скорочення (наприклад, `alias ll='ls -la'`).  
man команда — довідка про команду (вихід – `q`).  
команда --help — коротка підказка (працює майже для всіх команд).

## nano — простий і зрозумілий

`nano` завжди показує підказки внизу екрана. Комбінації записані через `^` (це клавіша Ctrl).

- **Відкрити файл:** `nano назва_файлу`  
  Якщо файл не існує — він створиться при збереженні.

- **Основні дії:**
  - `Ctrl+O` — зберегти файл (потім Enter для підтвердження імені).
  - `Ctrl+X` — вийти (якщо є незбережені зміни — запитає Y/N).
  - `Ctrl+W` — пошук тексту.
  - `Ctrl+K` — вирізати поточний рядок.
  - `Ctrl+U` — вставити вирізаний рядок.
  - `Ctrl+G` — довідка (усі комбінації).

Ніяких режимів — одразу можна друкувати текст, як у звичайному блокноті. Ідеально для новачка.

## vim — потужний, але з режимами

`vim` працює в кількох режимах. На старті ти в **нормальному режимі** (перегляд/навігація), для введення тексту потрібно перейти в **режим вставки**.

- **Відкрити файл:** `vim назва_файлу`

- **Найголовніше — як не застрягти:**
  - **Щоб почати писати** — натисни `i` (insert).
  - **Щоб вийти:**
    1. Натисни `Esc` (перейти в нормальний режим).
    2. Надрукуй `:q!` і Enter — вийти без збереження.
    3. Або `:wq` і Enter — зберегти та вийти.
  - **Якщо застряг і не розумієш, що відбувається** — натискай `Esc` кілька разів, потім `:q!`.

- **Базові команди нормального режиму (після Esc):**
  - `dd` — видалити рядок.
  - `yy` — скопіювати рядок.
  - `p` — вставити скопійоване/видалене.
  - `u` — скасувати останню дію (Ctrl+Z).
  - `Ctrl+R` — повернути скасоване.
  - `/текст` — пошук тексту (Enter для пошуку, `n` — наступний збіг, `N` — попередній).

- **Режим вставки (після `i`):** просто друкуєш текст. Повернутися в нормальний режим — `Esc`.

- **Збереження та вихід (у нормальному режимі):**
  - `:w` — зберегти.
  - `:q` — вийти (якщо збережено).
  - `:wq` або `ZZ` — зберегти і вийти.
  - `:q!` — вийти без збереження.

## Керування службами (сервісами) — `systemctl`

- **`systemctl start назва`** — запустити службу прямо зараз.  
  Приклад: `sudo systemctl start bluetooth`.

- **`systemctl stop назва`** — зупинити запущену службу.

- **`systemctl enable назва`** — дозволити службі автоматично запускатися при старті системи.  
  Приклад: `sudo systemctl enable lightdm`.

- **`systemctl disable назва`** — заборонити автозапуск (але не зупиняє, якщо служба зараз працює).

- **`systemctl restart назва`** — перезапустити службу (корисно після зміни конфігурації).

- **`systemctl reload назва`** — перечитати конфігурацію без повного перезапуску (не всі служби підтримують).

- **`systemctl status назва`** — показати статус служби: чи працює, PID, останні рядки логу.  
  Вийти з перегляду — `q`.

- **`systemctl is-enabled назва`** — перевірити, чи служба в автозапуску.

- **`systemctl is-active назва`** — перевірити, чи служба зараз працює.

## Перегляд усіх служб та стану системи

- **`systemctl list-units --type=service`** — список усіх запущених служб.

- **`systemctl list-units --type=service --state=running`** — тільки ті, що зараз працюють.

- **`systemctl list-unit-files --type=service`** — список усіх відомих служб (зокрема вимкнених) та їхній статус автозапуску.

- **`systemctl list-units --failed`** — показати служби, які впали (корисно для діагностики).

## Керування системою (перезавантаження, вимкнення, режими)

- **`systemctl reboot`** — перезавантажити комп’ютер.

- **`systemctl poweroff`** — вимкнути комп’ютер.

- **`systemctl suspend`** — приспати (сон, suspend to RAM).

- **`systemctl hibernate`** — глибокий сон (hibernate, збереження на диск).

- **`systemctl hybrid-sleep`** — гібридний сон.

Усі ці команди вимагають `sudo`, якщо виконуються не з root.

- **`systemctl list-timers`** — показати всі активні таймери (розклад завдань).

- **`systemctl enable --now назва.timer`** — увімкнути таймер та запустити його зараз.

## Перегляд логів — `journalctl`

Усі логи systemd збираються в журнал, доступний через `journalctl`.

- **`journalctl`** — показати всі логи (гортати стрілками, вийти `q`).

- **`journalctl -xe`** — показати логи з кінця з поясненнями помилок (ідеально для діагностики).

- **`journalctl -u назва.service`** — логи тільки для конкретної служби.

- **`journalctl -f`** — режим реального часу (як `tail -f`).  
  Зупинити — Ctrl+C.

- **`journalctl --boot=0`** — логи поточного завантаження.  
  `--boot=-1` — попереднє завантаження.

- **`journalctl --since "2026-06-28 10:00"`** — логи після вказаної дати.

## Керування часом, датою, локаллю

- **`timedatectl`** — показати поточний час, дату, часовий пояс.

- **`timedatectl list-timezones`** — список усіх часових поясів.

- **`sudo timedatectl set-timezone Europe/Kiev`** — змінити часовий пояс.

- **`sudo timedatectl set-time "YYYY-MM-DD HH:MM:SS"`** — встановити час вручну.

- **`hostnamectl`** — інформація про хост (назва комп’ютера, версія ядра).

- **`sudo hostnamectl set-hostname нове_ім’я`** — змінити ім’я комп’ютера.

- **`localectl`** — переглянути поточну локаль (мову, розкладку клавіатури).

## Аналіз завантаження та швидкодії

- **`systemd-analyze`** — показати, скільки часу завантажувалася система.

- **`systemd-analyze blame`** — список служб, відсортований за часом запуску (знайти “гальма”).

- **`systemd-analyze critical-chain`** — показати ланцюжок служб, який найбільше затримує завантаження.

## Повернення до пакетів (Arch)

- **`pacman -Q`** — список усіх встановлених пакетів.  
  `pacman -Q | grep назва` — знайти конкретний.

- **`pacman -Qi назва`** — детальна інформація про встановлений пакет.

- **`pacman -Ql назва`** — показати всі файли, що належать пакету.

- **`sudo pacman -Syu`** — повне оновлення системи (оновлює базу пакетів та всі пакунки).

  **`sudo pacman -Scc`** — повністю очистити кеш (звільняє місце, але не можна буде відкотити пакет).

## 1. `firewalld` — брандмауер (фаєрвол)

На відміну від багатьох дистрибутивів, в Arch за замовчуванням немає активного брандмауера.  
`firewalld` — простий у керуванні, використовує зони.

**Встановлення:**
```bash
sudo pacman -S firewalld
sudo systemctl enable --now firewalld
```

**Базові команди:**

- **`sudo firewall-cmd --state`** — перевірити, чи працює (має вивести `running`).

- **`firewall-cmd --list-all`** — показати поточну зону (зазвичай `public`) і її правила.

- **`sudo firewall-cmd --add-service=назва --permanent`** — дозволити відомий сервіс (наприклад, `http`, `https`, `ssh`).  
  Приклад: `sudo firewall-cmd --add-service=http --permanent`

- **`sudo firewall-cmd --add-port=порт/протокол --permanent`** — відкрити конкретний порт.  
  Приклад: `sudo firewall-cmd --add-port=3000/tcp --permanent`

- **`sudo firewall-cmd --remove-service=назва --permanent`** — прибрати сервіс.

- **`sudo firewall-cmd --reload`** — застосувати зміни без перезапуску служби (обов’язково після змін).

- **`sudo firewall-cmd --get-active-zones`** — показати активні зони та мережеві інтерфейси.

- **`sudo firewall-cmd --zone=public --change-interface=eth0`** — прив’язати інтерфейс до зони.

**Порада:** Після першого ввімкнення він блокує всі вхідні з’єднання, крім дозволених. Це нормально.

## 2. `cryptsetup` — шифрування дисків та розділів

Основа для LUKS (Linux Unified Key Setup) — стандартного шифрування в Linux. Часто використовується при встановленні Arch для шифрування кореневого розділу, але можна застосовувати й для флешок чи окремих розділів.

**Встановлення:** зазвичай уже є (пакет `cryptsetup`). Якщо ні:
```bash
sudo pacman -S cryptsetup
```

**Основні дії з LUKS-розділом (обережно — дані на розділі знищуються!):**

- **Створити шифрований розділ (перший раз):**  
  ```bash
  sudo cryptsetup luksFormat /dev/sdb1
  ```
  Запитає пароль (двічі). **Усі дані на sdb1 будуть втрачені.**

- **Відкрити шифрований розділ (примонтувати як пристрій):**  
  ```bash
  sudo cryptsetup open /dev/sdb1 mydata
  ```
  Тепер з’явиться пристрій `/dev/mapper/mydata`, з яким можна працювати як зі звичайним диском (створити файлову систему, монтувати).

- **Закрити (від’єднати):**  
  ```bash
  sudo cryptsetup close mydata
  ```

- **Додати додатковий ключ (наприклад, ключовий файл):**  
  ```bash
  sudo cryptsetup luksAddKey /dev/sdb1
  ```

- **Перевірити, чи розділ є LUKS-контейнером:**  
  ```bash
  sudo cryptsetup isLuks /dev/sdb1
  ```

- **Інформація про контейнер:**  
  ```bash
  sudo cryptsetup luksDump /dev/sdb1
  ```

**Поширений сценарій — шифрована флешка:**  
```bash
sudo cryptsetup luksFormat /dev/sdc1
sudo cryptsetup open /dev/sdc1 secretusb
sudo mkfs.ext4 /dev/mapper/secretusb
sudo mount /dev/mapper/secretusb /mnt
```

## 3. `makepkg` — збирання пакунків з AUR

`makepkg` — це команда для збирання пакунків Arch з вихідних кодів або скриптів `PKGBUILD`. Без неї неможливо користуватися AUR вручну (хоча `yay` робить це автоматично).

**Встановлення:** `makepkg` входить у пакет `base-devel`, який має бути встановлений у будь-якого розробника на Arch.  
Якщо його немає:
```bash
sudo pacman -S --needed base-devel
```

**Як використовувати (після клонування AUR-репозиторію):**

- **Зібрати та встановити пакет (найчастіше):**  
  У папці з `PKGBUILD` виконай:
  ```bash
  makepkg -si
  ```
  - `-s` — автоматично встановити залежності через pacman.  
  - `-i` — після збирання одразу встановити пакет.

- **Тільки зібрати, без встановлення:**  
  ```bash
  makepkg -s
  ```
  Готовий файл `.pkg.tar.zst` можна встановити потім командою `sudo pacman -U файл.pkg.tar.zst`.

- **Видалити всі тимчасові файли після збирання:**  
  ```bash
  makepkg -c
  ```

- **Перевірити цілісність PKGBUILD:**  
  ```bash
  makepkg --verifysource
  ```

**Важливо:** Ніколи не запускай `makepkg` від root! Працюй під звичайним користувачем. Після збирання пакет встановлюється через `sudo pacman -U`.

**Приклад повного циклу (ручна збірка з AUR):**
```bash
git clone https://aur.archlinux.org/назва-пакету.git
cd назва-пакету
makepkg -si
```
